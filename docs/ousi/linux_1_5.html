Scalix 功能特性

通透，全功能的 Outlook 支援
  • 整合企業既有的 MAPI/Outlook 及工作群組模式
  • 多種格式的電子郵件(Rich mail)：RTF 內容、附件、旗標、追蹤及送交等選項
  • 企業安排日程和預定時間表
  • 會議需求管理：邀請、接受、拒絕、延期至新時間等功能
  • 即時空閒/忙碌的行程查找(無須以Internet公告)
  • 資源預約
  • 個人聯絡人及郵件連絡人群組管理
  • 工作追蹤及委派
  • 個人資料夾
  • 公用資料夾
  • 委派存取行事曆及電子郵件
  • 離線作業及同步化
  • 公用連絡人群組
  • Client 及伺服器規則精靈功能
  • 支援Single Sign-on功能
  • 數位簽章
  • 支援Microsoft Outlook 2000、XP、2003
可靠度及適型化的實際應用驗證
  • 每台Scalix授權伺服器可支援超過5,000個企業電子郵件信箱
  • 99.99%以上的可靠度
  • 可依企業的大、中、小做規模增減
  • 線上備份
  • 各別訊息回復功能
  • 高可用度及支援主要領導的叢集解決方案 (A/A及A/P 模式，支援到8個 node)
  • 支援RAID 1 或 5 的磁碟備援
企業訊息系統架構
  • 適用不同未知的 Client 架構
  • 彈性、高效能的郵件路由
  • 每部伺服器支援數千企業內使用者的適型化訊息儲存
  • 可延伸擴建的企業目錄
  • 植基於標準的的開放介面
  • 發揮Linux平台的優點：可靠性、適型化、彈性化及發展的開放性
Scalix Web Access (Web client)
  • 可經由大部份知名的網頁瀏覽器(IE、FireFox)安全地存取電子郵件
  • 相同於桌面應應系統的可用性：拖拉、物件及選單三種格式介面
  • 多種格式的電子郵件(Rich mail)：HTML 內容、附件、拼字檢查、重要性及追蹤的選項
  • 企業用安排日程和預定時間表
  • 會議需求管理－邀請、接受、拒絕、延期至新時間等功能
  • 即時空閒/忙碌的行程查找(無須以Internet公告)
  • 個人連絡人管理
  • 個人資料夾
  • 公用資料夾
  • 專為提高大型資料夾及電子郵件信箱郵生產力的資料夾快取功能
  • 支援 web services 架構的標準基礎：XML、SOAP、DHTML、SSL、HTTP
  • 支援Internet Explorer 5、5 SP2、6、7
  • 支援Mozilla 1.7 on Linux、Windows and Mac OS X
  • 支援Firefox 1.0、1.5 on Linux、Windows and Mac OS X
  • 與Outlook及Evolution完全互通
Scalix Evolution連接模組(Scalix Evolution Connection)
  • 整合式訊息系統、行事曆系統及目錄服務
  • Rich Text Format(RTF)訊息格式
  • 離線電子郵件資料夾同步化(IMAP offline 模式)
  • 對於群組行事曆的空閒/忙碌目錄行程查找
  • 用戶端及伺服器規則精靈功能
  • 公用資料夾
  • 資料夾管理、儲存及搜尋
  • 公用郵件連絡人群組
  • 完全整合 Evolution 郵件簿及連絡人
  • 支援Novell Evolution 2.4.x
  • 支援Fedora Core 4 和 Red Hat Enterprise Linux 4
  • 可從 www.scalix.com 下載二進位檔和安裝檔
  • GNOME CVS 可獲得樹狀原始檔

  • IMAP 及 POP clients
    1. Outlook Express 5.5、6.0
    2. Eudora 5.2、6.x、7
    3. Mozilla Mail 1.7 和 Thunderbird 1.0、1.5
    4. SquirrelMail 1.4
遷移與共存
  • 針對Exchange及標準基礎電子郵件系統的最佳化遷移及方法.延伸至附加的專屬訊息系統平台
  • 全面性資料遷移(電子郵件、行事曆、連絡人、工作、記事、公用資料夾、公用郵件連絡人群組)
  • 全面化訊息應對能力及精確化
  • SMTP 電子郵件路由
  • 公用資料夾並存
  • 具遷移回復能力
  • 支援與現有存在的電子郵件系統並存—沒有“淘汰且取代”
  • 靈活的遷移時間軸，批次產生或手動
  • 目錄同步化與並存：
    1. Microsoft Active Directory 或 Exchange 5.5
    2. LDAP directories
防護
  • 可選擇的密碼強迫規則：期限、最小長度、重試次數等
  • 可選擇的非外部LDAP目錄認證
  • 單一登入(SSO) 及 Kerberos認證
  • 標準防垃圾郵件及內容過濾(RFC 2505)
  • 支援中立性的防毒功能
  • 經由存取控制表列來做公用資料夾的存取防護
  • SMTP 認證
  • POP、IMAP、LDAP 協定的通道加密
伺服器系統需求
  • 作業系統：
    1. Red Hat Enterprise Linux 3.0、4的 ES 或 AS 版本；SUSE Linux Standard Server 或Enterprise Server 9(x86 architecture)
    2. Fedora Core 4 及 SUSE Linux OSS 10.0(x86 architecture)
  • Scalix 執行於一般的 AMD/Intel Pentium IV 或 XeonCPU的伺服器 (一部2顆 CPU 的系統足以執行最大到5000個郵件信箱)
  • 磁碟儲存需求－Scalix 支援直接附掛及 SAN 儲存架構
  • 支援 IBM zSeries 大型主機(Red Hat Enterprise Linux 4 及 SUSE Linux Enterprise Server 9)
  • 如果做為商業營運用的重要關鍵電子郵件系統，Scalix建議兩部伺服器，做叢集架構：
    1. 每部使用雙處理器(Xeon or equiv) 並裝置 4GB 記億體
    2. 直接附掛儲存設備(DAS)、儲存區域網路(SAN)或網路附加儲存(NAS)，NAS 最好用於硬體快照，如果使用 DAS，必須是 RAID 1+0；如果使用 SAN/NAS 那麼 RAID 5 就足夠
    3. 支援叢集 Red Hat Cluster Suite、Novell SUSE Heartbeat 或 equivalent
